# Spec: 홈 탭 손익 섹션 소수점 제거

## 배경 / 문제

홈 탭의 3분할 손익 섹션(평가손익 / 확정손익 / 이달 확정)에서 백엔드가 보낸 손익 값에 소수점이 포함되면, `PnLText` 컴포넌트가 `toLocaleString("ko-KR")`을 통해 소수점까지 그대로 출력해 좁은 박스 영역을 벗어나는 현상이 있다.

- 현재 1천만 원 이상은 `fmtCompact()`로 압축 표기되지만, 1천만 원 미만이면 원본 숫자를 그대로 포맷하므로 `12,345.67원`처럼 소수점이 노출됨.
- 동일 유틸인 `formatPnL()`(format.ts:7-12)은 이미 `Math.round()`를 적용 중이라, `PnLText`만 일관성에서 벗어나 있는 상태.

## 목표

- 홈 탭 손익 3 섹션(평가손익 / 확정손익 / 이달 확정) 모두 정수원 단위로 표시되어 박스 영역을 벗어나지 않는다.
- 부호(+/−)와 색상, 1천만 이상 압축 표기(`fmtCompact`) 동작은 그대로 유지된다.

## 설계

### 접근 방식

`PnLText` 컴포넌트(`app/src/components/home/DashboardSummary.tsx:5-20`) 내부에서 입력값에 `Math.round()`를 적용해 정수로 변환한 뒤, 부호·절댓값·압축 분기를 수행한다.

- 변경 범위는 `PnLText` 한 곳으로 한정 — 다른 화면/컴포넌트는 영향 없음.
- `fmtCompact`/`fmt` 유틸 시그니처는 변경하지 않는다.
- 음의 작은 값(`-0.3`)이 `-0`으로 보이지 않도록, 반올림 후의 값으로 부호를 결정한다.

### 주요 변경 파일

- `app/src/components/home/DashboardSummary.tsx` — `PnLText`에서 `Math.round(value)`로 정수화 후 부호/절댓값/압축 처리 적용.

## 구현 체크리스트

- [x] `PnLText`에서 `value`를 `Math.round`로 정수화하고, 정수화된 값으로 부호·절댓값·`fmtCompact` 분기를 수행하도록 수정
- [x] 타입 체크 통과 (`pnpm -C app exec tsc --noEmit`)
- [ ] 홈 탭에서 평가손익/확정손익/이달 확정 세 섹션이 정수원으로만 표시되며 박스를 벗어나지 않는지 시각 확인 (가능하면 dev server)

## 우려사항 / 리스크

- 백엔드가 보낸 소수점 정보는 표시 단계에서만 버려지며 계산값에는 영향이 없다.
- 1억 이상은 여전히 `fmtCompact`가 `"5.2억"`처럼 1자리 소수점을 표시한다 — 사용자 선택(소수점만 제거)에 따라 이 압축 표기 자체는 유지한다.
