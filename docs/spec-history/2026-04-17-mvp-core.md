# MVP 코어 구현 완료 (Step 1–10)

완료일: 2026-04-17

## 구현 범위

### Step 1: 프로젝트 기초 세팅
- Supabase 패키지 설치 (`@supabase/supabase-js`, `@supabase/ssr`)
- Pretendard 폰트 적용
- 글로벌 CSS — 색상 변수 (상승 `#F04452`, 하락 `#1B6AC9`, 브랜드 `#3366FF`)
- `.env.local.example` 작성
- Supabase 클라이언트 유틸 (`src/lib/supabase/client.ts`, `server.ts`)
- 모바일 우선 뷰포트 메타 설정

### Step 2: DB 스키마 & 타입 정의

**마이그레이션 파일 5개:**

| 파일 | 내용 |
|------|------|
| `001_initial_schema.sql` | accounts/trades 테이블, Enum 타입, RLS, updated_at 트리거 |
| `002_add_trade_fees.sql` | `commission`, `tax` 컬럼 추가 |
| `003_fix_rls_update_check.sql` | RLS 업데이트 정책 수정 |
| `004_add_ticker_symbol.sql` | `ticker_symbol` 컬럼 추가 (nullable) |
| `005_add_country_code.sql` | `country_code` 컬럼 추가 (KR/US/OTHER, 기본값 'KR') |

**trades 테이블 전체 필드:**

| 필드 | 타입 | 비고 |
|------|------|------|
| id | uuid | PK |
| user_id | uuid | FK → auth.users |
| account_id | uuid | FK → accounts |
| asset_name | text | 종목명 |
| ticker_symbol | text | 종목코드 (nullable) |
| country_code | text | 'KR' / 'US' / 'OTHER' |
| market_type | MarketType | STOCK / CRYPTO / ETC |
| trade_type | TradeType | BUY / SELL |
| price | numeric | |
| quantity | numeric | |
| total_amount | numeric | 자동 계산 |
| traded_at | timestamptz | |
| strategy_type | StrategyType | |
| reasoning_tags | ReasoningTag[] | 다중 선택 |
| buy_reason | text | 매수 근거 |
| sell_reason | text | 매도 이유 |
| emotion | EmotionType | |
| result | TradeResult | SUCCESS / FAIL / BREAKEVEN |
| reflection_note | text | 회고 |
| improvement_note | text | 개선안 |
| profit_loss | numeric | SELL 시 실현 손익 |
| commission | numeric | 수수료 |
| tax | numeric | 세금 |

### Step 3: 인증
- 로그인/회원가입 페이지 (`src/app/login/page.tsx`)
- Supabase Auth 미들웨어 (세션 관리)
- 인증 상태 체크 & 리다이렉트
- 로그아웃 기능

### Step 4: 레이아웃 & 네비게이션
- 모바일 바텀 탭 네비게이션 4개 탭: 홈(`/`) / 기록(`/records`) / 분석(`/analysis`) / 설정(`/settings`)
- 앱 레이아웃 (`src/app/(app)/layout.tsx`) — 인증 보호

### Step 5: 계좌 관리
- 설정 페이지 (`src/app/(app)/settings/page.tsx`)
- 계좌 CRUD (추가/수정/삭제)
- 예수금 수동 입력 (`cash_balance`)
- 증권사 선택
- 계정 삭제 기능

### Step 6–7: 거래 기록 입력 폼
- 거래 입력 폼 (`TradeFormPanel`)
- 종목 자동완성 검색 (`StockSearchInput`)
- CSV 업로드 버튼 UI (`CsvUploadButton`) — 파싱/임포트 로직 미구현 (alert placeholder)
- 매수/매도 토글
- 가격/수량 입력 (자동 총액 계산)
- 날짜 선택, 계좌 선택
- 전략 버튼 그룹 (SCALPING / SWING / LONG_TERM / UNKNOWN)
- 감정 버튼 그룹 (CONFIDENT / ANXIOUS / FOMO / IMPULSIVE / CALM)
- 분석태그 다중 선택 (TECHNICAL / FUNDAMENTAL / NEWS / FEELING)
- 매수 근거 / 매도 이유 텍스트 입력 (선택)
- 수수료(`commission`), 세금(`tax`) 입력
- 폼 제출 → Supabase insert

### Step 8: 거래 목록 & 상세
- 기록 탭 — 거래 목록, 날짜별 그룹화
- 거래 카드 (매수/매도 뱃지, 전략/감정/결과 뱃지)
- 거래 상세 FullScreenPanel
- 거래 수정/삭제

### Step 8-부속: 종목 상세
- `/stocks/[country]/[ticker]` 페이지
- 특정 종목의 전체 거래 내역
- 성과 통계 (매도 횟수, 수익 횟수, 총 손익)

### Step 9: 복기 (회고)
- 매도 거래 메타: result(SUCCESS/FAIL/BREAKEVEN), reflection_note(회고), improvement_note(개선안)
- 기존 거래 수정 시 회고 추가 가능

### Step 10: 홈 대시보드

**`DashboardSummary` — 3개 통계 카드:**
- 평가손익 (미실현, 현재가 기준)
- 확정손익 (전체 실현 손익 합계)
- 이달 확정 (당월 실현 손익)

**`AllocationTabs` — 포트폴리오 도넛 차트:**
- 종목별 / 계좌별 탭 전환

**`HoldingsList` — 보유 종목 카드:**
- WAC(가중평균단가) 기반 평균단가
- 현재가, 평가금액, 손익률 표시
- 시세 출처: 네이버 금융(KR), Yahoo Finance(US)

## 시세 조회 구조 (`src/lib/quotes.ts`)

- `fetchKRPrice(ticker)` — 네이버 금융 비공식 API
- `fetchUSPrice(ticker)` — Yahoo Finance 비공식 API
- `fetchQuotesByKeys(keys)` — `"ticker:country"` 형식 배치 조회
- 실시간이 아닌 지연 시세 (공식 API 미연동)
