# Spec: 홈 대시보드 손익 카드 줄바꿈 방지

## 배경 / 문제

홈 탭의 `평가손익 / 확정손익 / 이달 확정` 3등분 카드(`grid-cols-3`)는 카드당 가용 폭이 매우 좁다(모바일 360px 기준 약 100px, 패딩 제외 73px). 현재 `PnLText`(`app/src/components/home/DashboardSummary.tsx:5-22`)가 `tabular-nums`만 갖고 있고 `whitespace-nowrap`이 없어, 금액이 커지면 카드 안에서 줄바꿈이 발생해 레이아웃이 깨지고 가독성이 떨어진다. 숫자는 어떤 경우에도 줄바꿈되어선 안 된다.

## 목표

- 평가손익/확정손익/이달 확정 카드의 금액이 절대로 줄바꿈되지 않는다.
- 큰 금액(천만 이상)도 카드 안에 안정적으로 표시된다.
- 작은 금액의 정확한 표시는 그대로 유지된다(1천만 미만은 콤마 풀 포맷).

## 설계

### 접근 방식

`PnLText`에 `whitespace-nowrap`을 추가하고, 절대값이 일정 임계값 이상이면 `fmtCompact`로 압축 표시한다. 부호(`+`/`-`)는 `PnLText`가 직접 붙이고, 값은 `Math.abs(value)`로 `fmtCompact`에 전달한다(현재 `fmtCompact`는 양수만 처리).

**임계값**: `|value| >= 10,000,000` (1천만)부터 압축
- `+1,234만원` (천만~억 미만)
- `+1.2억원` (1억 이상)
- 천만 미만은 기존처럼 `+123,456원` 풀 포맷 유지 → 일상적인 거래 손익은 정확도 보존.

### 주요 변경 파일

- `app/src/components/home/DashboardSummary.tsx` — `PnLText` 함수 수정:
  - 컨테이너 `<span>`에 `whitespace-nowrap` 클래스 추가
  - 표시 로직: `|value| >= 10_000_000` 이면 `fmtCompact(|value|) + "원"`, 아니면 기존 `fmt(value) + "원"`
  - 부호는 `value > 0 ? "+" : value < 0 ? "-" : ""`로 명시 출력

- `app/src/lib/format.ts` — 변경 없음. 호출부에서 `Math.abs` 처리.

## 우려사항 / 리스크

- 압축 표시는 반올림되므로 정확한 손익은 별도 화면에서 확인. 홈 카드는 요약 지표이므로 허용 가능.
- 1천만 임계값은 360px 기준 안전값이며, 시각 검증 후 필요시 조정.
